Chapter 0

先说结论：这个账号不是“装成博主”，而是训练一种稳定表达能力

账号本质 / 阅读方式

你要做的不是一个“会说金句的女生账号”，也不是一个“每天讲大道理的情感号”。

这个账号的核心，是让观众在你这里获得三种感觉：

  1. 她说出了我说不出来的东西。
  2. 她也不是完全强大的人，但她比我多往前走了一步。
  3. 我可以先不责怪自己，然后再做一点点具体的改变。

这个方向适合的账号气质是：女性成长、情绪自救、反内耗、真实生活记录、人生感悟口播、陪伴型表达。

你不是老师，不是心理医生，不是成功学导师，不是完美女主角。

你更像一个“互联网闺蜜”：她也会累、会嫉妒、会害怕、会拖延、会怀疑自己，但她愿意把混乱想清楚一点，再把这点清楚讲给别人听。

这个账号的真正生产方式，不是创作者一个人硬扛，而是三方协同：

  • 创作者：负责真实经验、出镜表达、个人判断、边界确认。
  • AI：负责对标分析、选题抽象、结构拆解、文案初稿、检查清单、复盘整理。
  • 工程师 / 主理人：负责系统、流程、素材采集、文件管理、工具自动化、版本记录、数据复盘、最终协调。

你只需要成为一个稳定提供真实感和镜头表达的人，不需要一个人学会所有采集、转写、剪辑、复盘、数据分析工具。

但是你也不能把自己完全交给 AI。AI 可以写稿，但不能替你活过这件事；AI 可以总结方法，但不能替你拥有态度；AI 可以做流程，但不能替你决定哪些话你真的愿意说。

本白皮书的目标：把已经跑通的一批成功对标视频，拆成适合新手执行的账号方法、三方分工、工作流程、内容标准和第一阶段行动计划。

开始正式写脚本之前，需要先做一次“创作者刻写”。创作者刻写不是编人设，而是把她真实的生活阶段、情绪经验、表达边界、说话方式和可公开素材整理出来。创作者本人不需要翻项目文件；她只需要读这份白皮书，并按第 19 章的模板回答问题。后台可以把答案同步整理到项目资料里，后续每条脚本都应该尽量连接到这些真实材料，否则内容很容易变成泛泛的 AI 女性成长文案。

给新手创作者的阅读方式：

  • 如果你只想知道这个账号是什么，先读第 0、3、5 章。
  • 如果你不知道自己该讲什么，读第 4、6、19.1、19.6 章。
  • 如果你想马上开始拍，读第 7、8、9、10、12、20 章。
  • 如果你担心 AI 写得不像你，重点读第 18、19.4、19.6 章。
  • 如果你只想执行第一周，直接看第 16 章和第 20 章。

你不需要知道项目里每个文件在哪里。工程师 / 主理人会维护后台文件。你只需要完成三件事：提供真实素材，判断稿子像不像你，完成拍摄和反馈。

Chapter 1

我们从已跑通视频里学到的核心规律

对标成功规律

本项目已处理的对标来源是抖音账号“鹿冰冰”的一批高互动口播 / 生活表达 / 女性成长视频。已完成的本地材料包括视频元数据、ASR 转写、部分深度分析、互动指标、视觉分析和生产标准。

注意：这些视频是学习对象，不是复制对象。我们学习的是结构、情绪推进、观众心理、选题判断、拍摄节奏；不能复制她的个人故事、原句、独特比喻、生活经历、文学引用链和表达习惯。

这批视频能跑出来，不是因为它们都短、都精致、都像营销号。恰恰相反，很多视频很长，画面很简单，甚至带有强烈的私人叙述。但它们有几个共同点：

  1. 不是先讲道理，而是先把一个具体痛感讲出来。
  2. 不是装作正确，而是先承认一个“不好看”的自己。
  3. 不是用空话安慰观众，而是给观众一种新的解释方式。
  4. 不是只有情绪，而是有生活细节、行为证据和因果链。
  5. 不是每条都追求点赞，而是分别触发评论、转发、收藏。
  6. 不是用复杂画面撑内容，而是用口播里的思考密度撑内容。

从数据形态看，这批视频大致有三种成功类型：

一类是“收藏型内容”。

特点：观众觉得这段话可以留给自己以后再看。

常见信号：收藏 / 点赞比例高。

内容特征：解释了一个长期困扰她的问题，提供了更温柔、更清楚的自我理解。

适合新手学习：强烈适合，因为它不依赖争议和表演，依赖真实表达和结构。

一类是“转发型内容”。

特点：观众想把这条发给别人，或者用它表达自己的立场。

常见信号：分享 / 点赞比例高。

内容特征：说出了一个群体共识、关系冲突、社会处境，或者一个“你看，我就是这个意思”的解释。

适合新手学习：适合，但要小心不要变成攻击别人或制造矛盾。

一类是“评论型内容”。

特点：观众忍不住参与判断、反驳、开玩笑、站队。

常见信号：评论 / 点赞比例高。

内容特征：有一个可被讨论的现场，例如回应批评、展示穿搭、表达争议观点。

适合新手学习：可以少量尝试，但新账号不要一开始就过度依赖争议。

这批视频告诉我们：女性成长口播不等于“温柔鸡汤”。真正有传播力的女性成长内容，通常是“温柔外壳 + 清醒骨架 + 具体伤口 + 一点点反抗”。

Chapter 2

对标视频中的可学习成功模式

可学习内容模式

以下模式来自已处理视频的结构抽象。它们只能作为创作方法，不能照搬原视频细节。

2.1 被羞耻的冲动 -> 具体伤口 -> 保护性重述

代表类型：把“贪婪、嫉妒、虚荣、想被看见、害怕普通、想要更多”这些不体面的感觉，重新解释成一个人过去受伤后的保护方式。

它为什么有效：

  • 观众不是没有这些情绪，而是不好意思承认。
  • 视频先替她承认，再替她找到更深层的原因。
  • 不是洗白，而是让她停止粗暴地审判自己。

结构可以这样用：

  1. 开头先承认一个不好看的自己。

例如：我发现我有时候真的很嫉妒别人。

  1. 给一个具体场景。

例如：刷到同龄人升职、恋爱、买车、旅行，我明明不认识她，却突然很难受。

  1. 不急着讲道理，继续追问。

我到底是在嫉妒什么？是她拥有了那些东西，还是她好像比我更有资格幸福？

  1. 找到伤口。

也许是我太久没有感到自己的生活在向前走。

  1. 给出保护性重述。

嫉妒不一定说明我坏，它有时候是在提醒我：我也有想要的东西，但我一直没敢承认。

  1. 结尾给一个小动作。

今天先不骂自己，先写下我真正想要的那件事。

新手注意：

  • 不要把负面情绪包装成“我就是高级”。
  • 不要把所有问题都归因于童年、原生家庭或创伤。
  • 不要编造严重经历。
  • 最后一定要落回“我能做一个小动作”，否则会变成情绪发泄。

2.2 生活小事故 -> 情绪爆炸 -> 人生解释

代表类型：从一件小事切入，比如外卖坏了、赶火车、窗帘透光、遛狗、洗头、打扫、医院陪护，然后讲出一个更大的生命感受。

它为什么有效：

  • 小事真实，观众容易进入。
  • 情绪不是凭空来的，有触发点。
  • 最后升维时，观众会觉得“原来我也不是因为这件小事才崩溃，而是很多东西一起压过来了”。

结构可以这样用：

  1. 开头说一个非常具体的小事。
  2. 描述当时身体 / 情绪反应。
  3. 追问：为什么这件小事让我这么难受？
  4. 扩展到长期状态：累、焦虑、失控、无力、怕麻烦别人。
  5. 给出新的理解：不是这件事有多大，而是我已经撑了很久。
  6. 结尾不要喊口号，只给许可：有时候崩溃不是矫情，是身体在提醒我停一下。

新手注意：

  • 小事必须具体，不能说“最近很累”就结束。
  • 不要一上来就抽象，先让观众看见你在哪、发生了什么。
  • 结论要从故事里长出来，不要强行拔高。

2.3 合理选择 -> 失去自己 -> 美化没选的路

代表类型：很多人生选择不是错的，甚至是当时最安全、最理性、最被认可的选择。但它仍然让一个人失去了某种可能性，于是她开始反复想象“如果当初不是这样”。

它为什么有效：

  • 很多女生的人生不是被一个坏人毁掉，而是被一连串“为你好”“更稳定”“更现实”的选择慢慢改道。
  • 观众会在里面看到自己的专业、工作、城市、恋爱、家庭选择。
  • 它没有简单责怪父母或社会，而是讲清楚“我为什么会一直惦记另一条路”。

结构可以这样用：

  1. 开头说一个曾经的原始愿望。

我以前真的很想做某件事。

  1. 出现一个外部合理建议。

但所有人都说另一条路更稳、更对、更有前途。

  1. 你照做了，或者半推半就地放弃了。
  2. 讲后果：不是立刻毁掉，而是慢慢没劲、慢慢失去支撑。
  3. 承认复杂性：我不能简单怪谁，因为那时候大家也只是想安全。
  4. 得出洞察：人会美化没选的那条路，不是因为它一定更好，而是因为现在这条路也很疼。
  5. 结尾落到现在：我现在能做的，不是回到过去，而是把一点选择权拿回来。

新手注意：

  • 不要写成单纯控诉家人。
  • 不要把没选的路说成一定成功。
  • 核心是“选择权”和“自我支撑”，不是后悔文学。

2.4 批评 / 质疑 -> 现场证明 -> 自嘲式反转

代表类型：别人说你穿搭土、表达不行、想法太多、不适合做账号，你不直接辩解，而是把这个质疑变成一场现场展示。

它为什么有效：

  • 它有互动感，观众像在围观一个现场。
  • 它有可视化证据，不只是嘴上反驳。
  • 它允许创作者显得有点可笑，但这种可笑会增加亲近感。

结构可以这样用：

  1. 开头引用一个批评。

最近有人说我……

  1. 表面说“我不是要吵架”，实际进入展示。
  2. 给 3-5 个证据，每个证据都比上一个更有戏。
  3. 中途允许自嘲，不要端着。
  4. 最后反转：这件事其实说明了我是什么样的人，或者说明我为什么不该只被这个评价定义。

新手注意：

  • 不要无中生有编造“很多人骂我”。如果没有真实评论，就说“如果你也担心别人会这样说”。
  • 不要把视频做成纯吵架。
  • 这个模式最好有画面证明：衣服、书桌、脚本、失败素材、拍摄过程、手写稿、剪辑界面等。

2.5 普通人定位 -> 放弃装高级 -> 找到可持续路线

代表类型：普通创作者最容易犯的错，是一有一点成绩就想把自己做得更“高级”、更像行业头部，最后反而变成四不像。

它为什么有效：

  • 新人创作者很容易共鸣：想做出调性，但没有足够资源、阅历和稳定表达。
  • 它指出了一个反直觉事实：普通人不是和行业顶端竞争，而是先和优秀普通人竞争。
  • 它能形成很强的收藏价值，因为观众会把它当成做事提醒。

结构可以这样用：

  1. 开头指出一个普通人常见错误。

很多人刚开始做账号，最危险的不是不努力，而是太早想显得高级。

  1. 给自己的例子或观察。
  2. 拆穿幻想：你不是在和顶级博主竞争，你是在找到一个自己能长期讲下去的位置。
  3. 讲代价：太早装高级，会丢掉真实、亲近、可持续表达。
  4. 给执行建议：先把你能讲清楚、能拍出来、能持续 30 条的内容做好。

新手注意：

  • 这类内容很适合你本人，因为你正处于启动阶段。
  • 但不要假装自己已经成功，要明确说“这是我刚开始时提醒自己的”。

2.6 知识 / 书 / 音乐 -> 个人命运感 -> 重新解释生活

代表类型：从一本书、一首歌、一个文学人物、一句歌词进入，但最后不是做书评，而是讲它怎样照见了自己的处境。

它为什么有效：

  • 文化内容给视频增加质感。
  • 但真正打动人的不是知识点，而是“这个知识点为什么和我的生活有关”。
  • 收藏率往往较高，因为观众觉得获得了一种表达语言。

结构可以这样用：

  1. 不要开头就像课堂。
  2. 先说“我最近被一句话 / 一首歌 / 一个故事击中了”。
  3. 解释它击中你的具体原因。
  4. 把它放回现实生活：工作、身体、家庭、选择、表达、关系。
  5. 结尾给观众一个可带走的理解，而不是推荐清单。

新手注意：

  • 只讲你真的读过、听过、理解过的东西。
  • 不要为了显得有文化而堆作者名字。
  • 不要把别人的文学表达当成自己的金句。

2.7 低能量 / 身体限制 -> 仍然把事办完 -> 重定义勇敢

代表类型：承认自己不是高能量、自律爆棚、每天状态很好的人，但自己仍然在用一种笨拙的方式把事情推进。

它为什么有效：

  • 现在很多年轻女性不是缺目标，而是被身体、睡眠、情绪、工作压力拖住。
  • 她们不需要被骂醒，需要被准确看见。
  • “我不是不努力，我只是成本比别人高”是强共鸣点。

结构可以这样用：

  1. 先承认限制：我不是那种精力很多的人。
  2. 讲具体表现：睡不好、容易累、拍一条视频要缓很久、出门也费劲。
  3. 不卖惨，讲真实成本。
  4. 重定义勇敢：不是每天元气满满，而是状态不好时仍然完成一点点。
  5. 给低门槛行动：今天只完成一个最小动作。

新手注意：

  • 涉及疾病、心理健康、医疗判断时必须谨慎，不要给别人诊断。
  • 可以说自己的感受，不要给观众提供医学建议。
Chapter 3

这个新账号应该怎么定位

账号定位

根据项目现有定位文件，这个账号的方向不是“单纯情感博主”，而是：

女性成长博主 / 互联网闺蜜 / 陪伴型表达者。

她服务的人群是：

  • 18-35 岁女性。
  • 学生、刚毕业、年轻职场女性。
  • 情绪内耗、关系受伤、家庭压力、自我怀疑的人。
  • 喜欢真实、故事型、不油腻、不鸡汤表达的人。
  • 想变好，但不想再被“你必须强大”压迫的人。

账号内容承诺：

看完之后，观众应该觉得：

  • 我不是一个人这样。
  • 我没有想象中那么糟糕。
  • 我可以稍微清楚一点。
  • 我今天不用立刻逆袭，但可以少攻击自己一点。
  • 我也许可以做一个很小的行动。

账号不能变成：

  • 情感导师号。
  • 心理治疗号。
  • 成功学号。
  • 纯穿搭 / 颜值号。
  • 纯读书号。
  • 高高在上的女性主义说教号。
  • 靠抄爆款金句堆出来的 AI 口播号。

账号的核心声音：

  • 直接，但不凶。
  • 清醒，但不审判。
  • 柔软，但不软弱。
  • 真诚，但不把伤口当表演。
  • 有思考，但不像论文。
  • 像朋友发来的语音，不像老师上课。

最重要的一句话：

你不是去“教育观众”，你是把你自己也在经历的混乱，整理成一个能让别人少痛苦一点的表达。

Chapter 4

新手创作者必须建立的内容资产

创作者刻写与素材库

不要每天坐在那里问“我今天发什么”。那样很快会枯竭。

你要建立的是四个内容资产库。

在建立这四个库之前，先做一次“创作者刻写”。这一步最重要，因为它决定后面的内容像不像你。

4.0 创作者刻写：先把人写清楚，再写内容

创作者刻写不是包装人设，也不是编一个完美女主角。

它要解决的问题是：

  1. 这个账号里的“我”是谁？
  2. 哪些经历可以公开讲？
  3. 哪些感受是我真的有的？
  4. 哪些表达我说出口会假？
  5. 哪些话题既能吸粉，又不会让我消耗过度？
  6. 观众为什么要关注我，而不是只看完这一条就走？

人设要分三层：

| 层级 | 含义 | 用途 |

|---|---|---|

| 真实的人 | 真实经历、生活阶段、性格、限制 | 原创性来源 |

| 公开的人设 | 观众能稳定记住的你 | 账号标签 |

| 每条视频里的角色 | 这条内容里你怎么说话、站在哪里 | 脚本执行 |

这三个层级不能打架。

如果一条稿子只符合“公开人设”，但真实的你根本说不出口，它就要改。

如果一条稿子来自真实经历，但暴露太多隐私、你还承受不了，它也不能直接发。

当前账号的人设假设可以先这样理解：

一个还没有完全变好、也不想装作强大的女生，正在学习停止责怪自己。她像一个互联网闺蜜，说话柔软、疲惫但清楚，比观众多往前走了一小步。

这个账号可以展示：

  • 害怕开始，但还是开始。
  • 低能量，但还想把生活拉回来。
  • 会嫉妒、会比较、会羞耻，但愿意诚实看见。
  • 在关系里受过消耗，但不想变得刻薄。
  • 是普通女孩，但不想一辈子只做“懂事的人”。
  • 想吸粉、想赚钱、想变好，但不想用假人设换流量。

这个账号不要展示：

  • 我已经完全好了。
  • 我来教你人生。
  • 我比你清醒。
  • 只要照做就能变好。
  • 为了流量讲自己还没准备好公开的伤口。

创作者刻写要收集 8 类材料：

  1. 生活阶段：现在在上学、工作、待业、转型、做账号，还是处在混乱期？
  2. 核心压力：最近最常困扰你的是什么？钱、关系、工作、外貌、家庭、未来、表达？
  3. 情绪惯性：你最容易陷入哪种情绪？自责、内耗、嫉妒、羞耻、拖延、讨好？
  4. 真实场景：最近哪些小事让你难受、清醒、崩溃或突然想明白？
  5. 关系模式：你在哪些关系里最容易失去自己？
  6. 欲望与羞耻：你想要什么，但不好意思承认？
  7. 表达习惯：你平时怎么说话？哪些词你说着自然，哪些词很假？
  8. 公开边界：哪些能讲，哪些要模糊处理，哪些绝对不讲？

这些材料最后会变成：

  • 真实经历库。
  • 选题库。
  • 开头库。
  • 不要说清单。
  • 可复刻的人设表达。
  • 以后脚本里的真实细节。

所以，前期最重要的不是马上写 100 条脚本，而是先把“这个人到底能讲什么”写清楚。

4.1 真实经历库

每天记录 5 类事情：

  1. 今天我在哪一刻突然不舒服？
  2. 今天我嫉妒、羡慕、羞耻、委屈、愤怒、害怕了什么？
  3. 今天有什么很小的生活细节让我想了很久？
  4. 今天我对自己说了什么难听的话？
  5. 今天有没有一个瞬间，我比昨天更清楚了一点？

记录不用优美，越具体越好。

差的记录：

  • 今天很累。
  • 最近很焦虑。
  • 想做账号但害怕。

好的记录：

  • 今天准备拍视频，打开前置摄像头看到自己的脸，突然觉得自己不配被别人看见。
  • 看到同龄女生发旅行照片，我不是想旅行，我是突然觉得她好像有一种“人生已经开始了”的感觉，而我还在原地。
  • 本来只想拍一条 30 秒视频，结果改稿 2 小时，最后什么都没拍，我开始怀疑自己是不是只会准备不会开始。

4.2 观点库

观点不是金句。观点是你对一件事的判断。

格式：

  • 我以前以为：
  • 现在我觉得：
  • 这个变化来自：
  • 我不确定的是：

例子：

  • 我以前以为做账号要先把自己变得很完美。
  • 现在我觉得，也许账号真正需要的是持续出现，而不是完美出现。
  • 这个变化来自我发现自己一直在用准备逃避被看见。
  • 我不确定的是，观众能不能接受我这么不成熟的阶段。

4.3 场景库

场景是口播的证据。

你要收集：

  • 镜子前。
  • 地铁上。
  • 工位上。
  • 晚上躺床上。
  • 和朋友聊天后。
  • 刷朋友圈后。
  • 拍视频失败后。
  • 改稿到崩溃后。
  • 收到一句评价后。
  • 看到一本书 / 一首歌 / 一条评论后。

每条视频至少需要一个真实场景，否则很容易变成空话。

4.4 对标模式库

不是收藏“好句子”，而是收藏“结构”。

记录格式：

  • 对标视频 ID / 链接：
  • 它讲的不是表面话题，而是：
  • 它的开头怎么抓人：
  • 它中间用了什么证据：
  • 它最后怎样转弯：
  • 我不能复制的部分：
  • 我可以学习的结构：
  • 我自己的版本可以换成什么经历：

这部分由 AI 和工程师负责主要整理，创作者只需要参与判断：这个结构我能不能说？这个情绪我是不是真的有？

Chapter 5

一条合格视频的标准

内容标准与阶段策略

一条合格的新手视频，不是看起来像爆款，而是同时满足以下条件：

  1. 开头 3 秒有真实钩子。
  2. 中间至少有一个具体场景或例子。
  3. 有一个从“情绪”到“理解”的转弯。
  4. 结尾给观众许可或小动作，而不是命令。
  5. 这段话创作者真的能说出口。
  6. 没有复制对标账号的独特表达。
  7. 拍摄难度在当前资源下可完成。
  8. 发布后能被复盘，而不是发完就算。

项目内容标尺使用 7 个维度：

  • HF Hook Force：开头有没有留人理由。
  • PF Persona Fit：像不像这个创作者本人。
  • OA Original Angle：有没有自己的角度，而不是改写别人。
  • AV Audience Value：观众看完是否更清楚、更勇敢、更能行动。
  • CE Concrete Evidence：有没有故事、场景、细节、证据。
  • SF Shootability Fit：现在能不能拍好。
  • CR Conversation Risk/Reward：能不能引发有价值讨论，同时风险可控。

冷启动阶段不要迷信分数。分数只是帮助判断，不是装专业。真正重要的是：这条能不能拍？拍出来像不像你？观众能不能得到一个清楚的东西？

5.1 内容判断必须分阶段：前期起号，中期信任，后期变现

这个账号的最终目的不是“写出漂亮文案”，而是让创作者获得真实增长：吸粉、沉淀信任、自然引流，并最终形成能挣钱的内容资产。

但是不同阶段不能用同一套标准判断内容。

前期最重要的是快速起号；中期最重要的是稳定信任；后期最重要的是商业承接。如果冷启动一开始就按后期变现标准做内容，常见结果是：内容太重、太像销售、没有人愿意先关注你。如果后期还停留在纯情绪共鸣，常见结果是：有点赞、有共情，但账号不挣钱。

阶段目标可以这样理解：

| 阶段 | 核心目标 | 内容任务 | 不要过早追求 |

|---|---|---|---|

| 前期 | 快速起号 | 拉新、涨粉、打标签、测试爆点 | 强转化、卖课、复杂人设 |

| 中期 | 稳定信任 | 系列化、观点沉淀、软引流、评论关系 | 只追热点、只做情绪爆款 |

| 后期 | 引流变现 | 产品承接、案例证明、社群/咨询/品牌合作 | 无目的涨粉、无承接共鸣 |

这三个阶段的共同底层逻辑是：

  1. 前期让陌生人知道“她懂我”。
  2. 中期让老观众知道“她值得我反复回来”。
  3. 后期让高信任观众知道“我可以进一步找她解决问题 / 购买相关产品 / 进入她的服务场景”。

所以，一条视频值不值得复刻，不只看原视频数据好不好，还要看它适合哪个阶段。

5.2 前期：快速起号的内容逻辑

前期的唯一核心任务是：让目标女生在 3-5 秒内觉得“她说的是我”，并且愿意关注这个账号继续听。

前期不适合一上来强引流、强销售、强专家人设。这个阶段应该优先做“高命中、低门槛、强标签”的内容。

前期最有效的起号视频，通常有 6 类：

  1. 痛点命名型

核心句式：你不是___，你只是___。

作用：帮观众说出她说不清的痛苦。

例子：你不是太敏感，你只是太久没有被好好对待过。

  1. 反自责型

核心句式：以前我也以为是我___，后来才发现，其实是___。

作用：把观众正在责怪自己的事情，换一个更温柔、更清醒的解释。

  1. 关系清醒型

核心句式：真正消耗你的关系，不是___，而是___。

作用：触发评论、转发和“我也是”的关系共鸣。

  1. 普通女孩 / 小镇女孩共鸣型

核心句式：有些女孩不是不___，是她从小就没有被允许___。

作用：形成差异化，不只做泛泛女性成长。

  1. 生活瞬间顿悟型

核心句式：我是在___的时候，突然明白___。

作用：用真实日常建立陪伴感，避免像营销号。

  1. 低能量自救型

核心句式：如果你现在真的很累，先别急着___，你可以先___。

作用：让低能量观众被接住，提升收藏和关注。

前期建议的 30 条内容配比：

| 类型 | 条数 | 作用 |

|---|---:|---|

| 痛点命名型 | 8 | 快速打标签，拉精准人群 |

| 反自责型 | 6 | 建立陪伴感和信任 |

| 关系清醒型 | 5 | 拉互动和评论 |

| 普通女孩 / 小镇女孩共鸣型 | 4 | 做差异化 |

| 生活瞬间顿悟型 | 4 | 建立真实人设 |

| 低能量自救型 | 3 | 增强收藏和陪伴感 |

前期不要用“能不能马上变现”作为最高标准。前期真正要问的是：这条能不能让精准女生先关注我？

5.3 前期起号六芒星

前期六芒星不评价“这条内容高不高级”，只评价“这条能不能帮助起号”。

| 维度 | 核心问题 |

|---|---|

| 秒懂吸粉力 | 陌生人 3 秒内能不能知道这条和她有关？ |

| 痛点命中力 | 有没有说中目标女生的具体处境？ |

| 关注理由 | 看完会不会觉得“我想继续听她说”？ |

| 账号标签力 | 这条能不能强化女性成长 / 情绪自救 / 互联网闺蜜定位？ |

| 互动传播力 | 有没有评论、转发、收藏的理由？ |

| 低成本复刻力 | 能不能 30-90 秒口播快速拍出来？ |

每项 0-5 分：

  • 0 = 完全不成立，甚至反向伤害账号。
  • 1 = 很弱，只能参考。
  • 2 = 有一点价值，但不建议优先。
  • 3 = 可用，可以进入测试池。
  • 4 = 强，值得优先做。
  • 5 = 极强，是起号主力。

前期起号综合分：

前期起号分 =
秒懂吸粉力 * 25%
+ 痛点命中力 * 25%
+ 关注理由 * 20%
+ 账号标签力 * 15%
+ 互动传播力 * 10%
+ 低成本复刻力 * 5%

前期决策标准：

| 分数 | 结论 | 动作 |

|---:|---|---|

| 85-100 | 起号主力 | 马上出脚本，优先拍 |

| 70-84 | 值得测试 | 进入前 30 条选题池 |

| 55-69 | 可拆局部 | 只学习钩子、结构或表达方式 |

| 40-54 | 暂存观察 | 不作为冷启动主线 |

| 0-39 | 跳过 | 不复刻 |

硬门槛：

  • 秒懂吸粉力 < 3：前期不做。
  • 痛点命中力 < 3：前期不做。
  • 账号标签力 < 3：即使原视频数据好，也只参考，不作为主线。
  • 关注理由 >= 4 且痛点命中力 >= 4：优先复刻。

5.4 中期：从“她懂我”到“我信她”

中期通常发生在账号已经有一批稳定关注者以后。这个阶段不能只追求每条都情绪命中，而要开始沉淀“我为什么值得被长期关注”。

中期内容要完成三件事：

  1. 系列化：让观众知道她下次还能在你这里看到什么。
  2. 信任化：让观众知道你不是只会说情绪金句，而是真的有观察、有过程、有边界。
  3. 软引流：让观众自然知道你可以提供什么帮助、陪伴、资料、社群或产品方向。

中期适合增加的内容：

  • 连续系列：例如“普通女孩做账号记录”“我也是后来才明白”“低能量自救日记”。
  • 过程透明：展示写稿、改稿、拍摄失败、复盘、练习表达。
  • 评论回应：从真实评论里挑选高频困境进行回应。
  • 轻承接内容：例如资料清单、打卡计划、陪伴社群、读书/成长清单，但不强卖。

中期六芒星可以调整为：

| 维度 | 核心问题 |

|---|---|

| 信任建立力 | 这条能不能让观众觉得你真实、有判断、有边界？ |

| 系列资产力 | 这条能不能发展成可持续栏目？ |

| 复访理由 | 观众看完会不会想下次继续回来？ |

| 私域入口感 | 有没有自然产生“我想要更多”的入口？ |

| 人设加深度 | 有没有让创作者从“懂我”变成“我认识她”？ |

| 数据延展力 | 能不能从评论、收藏、转发里长出下一批选题？ |

中期不应该突然变成销售号。中期的商业动作要轻，重点是让观众从“关注你”过渡到“信任你”。

5.5 后期：从“我信她”到“我愿意付费 / 合作”

后期的目标才是更明确的引流和变现。这个阶段不是把每条视频都做成广告，而是让内容和产品承接之间有清楚关系。

后期内容要回答：

  1. 你能帮谁？
  2. 你能帮她解决什么具体问题？
  3. 为什么是你？
  4. 她下一步应该去哪里？
  5. 这个承接是否不伤害账号信任？

后期可发展的商业方向包括：

  • 女性成长 / 情绪自救陪伴社群。
  • 低能量自救打卡计划。
  • 读书、表达、写作、账号启动类轻产品。
  • 美妆、服饰、生活方式、书籍等品牌合作。
  • 轻咨询或陪伴型服务，但必须注意边界，不承诺治疗或结果。

后期六芒星可以调整为：

| 维度 | 核心问题 |

|---|---|

| 付费人群贴合度 | 这条吸来的人是不是未来可能付费 / 合作的人？ |

| 需求明确度 | 观众看完是否更清楚自己需要什么？ |

| 信任证明力 | 有没有案例、过程、真实反馈或长期积累支撑？ |

| 引流自然度 | 引导私域 / 产品 / 咨询是否自然、不生硬？ |

| 变现安全度 | 是否避免夸大承诺、心理医疗化、结果保证？ |

| 规模复制力 | 这类内容能不能长期生产并服务商业目标？ |

后期判断一条视频值不值得做，不能只看播放。更重要的是：它吸来的人对不对，能不能进入下一步关系。

5.6 阶段切换规则

不要用单一粉丝数机械切换阶段。更稳妥的判断方式是看内容能力有没有出现重复信号。

可以进入中期的信号：

  • 有 3 条以上视频证明某类痛点能稳定吸引目标观众。
  • 评论里开始出现“我也是”“你说到我了”“想继续听”这类稳定反馈。
  • 创作者能稳定说出 2-3 个栏目方向，而不是每次重新找感觉。
  • 账号标签已经比较清楚：观众知道你大概讲什么。

可以进入后期的信号：

  • 有一批观众持续互动，而不是只看单条爆款。
  • 评论或私信里开始出现明确需求：想要清单、方法、陪伴、咨询、社群、产品推荐。
  • 创作者已经知道自己能提供什么，不能提供什么。
  • 至少有一个轻承接产品或私域入口，不会伤害账号信任。

如果没有这些信号，不要急着卖。前期先起号，中期先稳信任，后期再承接商业。

Chapter 6

选题怎么做：不要追热点，要抓“隐形痛点”

选题方法

新手最容易犯的错，是看到什么火就拍什么。这个方向不能这么做。

女性成长 / 自愈口播的选题，应优先来自这些来源：

  1. 你最近真实经历过的情绪。
  2. 对标视频中反复出现的高共鸣结构。
  3. 观众评论里反复出现的困境。
  4. 你身边女生常说但说不清的问题。
  5. 你自己过去一直误解自己的地方。
  6. 你正在做账号这件事本身带来的羞耻、害怕、拖延、比较。

高潜力选题通常长这样：

  • 不是“如何变自律”，而是“你不是不自律，你只是一直在用准备逃避开始”。
  • 不是“女生要独立”，而是“有时候你不敢要，是因为你太怕别人看见你真的想要”。
  • 不是“停止内耗”，而是“你脑子里不是废话多，是你从来没被允许把话说完整”。
  • 不是“做自己”，而是“做自己之前，你可能要先承认自己很怕被讨厌”。
  • 不是“普通女孩也能逆袭”，而是“普通女孩先别急着逆袭，先稳定出现”。

选题筛选的 5 个问题：

  1. 这是不是目标观众真实会遇到的问题？
  2. 我有没有自己的场景或经历能支撑它？
  3. 它有没有一个反常识或重新解释的角度？
  4. 它会不会变成空泛鸡汤？
  5. 它现在能不能拍？

不建议新手一开始做的选题：

  • 需要大量专业背书的心理学 / 医学 / 法律建议。
  • 过度宏大的社会议题。
  • 纯粹攻击男性、父母、职场、婚姻的情绪内容。
  • 明显照搬对标账号个人经历的题。
  • 自己没有真实经验却硬讲的深刻话题。
  • 纯热点跟风。
Chapter 7

开头怎么写：第一句要像“突然被看见”

开头写法

这个账号的开头，不适合传统营销号那种：

  • “女生一定要知道的三件事。”
  • “如果你想变强，请看完这条视频。”
  • “真正厉害的女人都明白……”

更适合的开头，是把观众心里一个隐蔽感受直接说出来。

好开头的类型：

  1. 自我承认型
  • 我发现我有时候不是不努力，我是太害怕被看见了。
  • 我以前一直以为我是懒，后来才发现我是在逃避失败后的羞耻。
  • 我最近才承认，我其实很嫉妒那些看起来很平静的人。
  1. 反解释型
  • 有时候你不是想太多，你只是太久没有把话说完整。
  • 你不是没有行动力，你可能只是把开始想得太神圣了。
  • 你不是玻璃心，有些阶段活着本来就很费力。
  1. 具体场景型
  • 今天我打开前置摄像头的那一秒，突然不想做账号了。
  • 我昨晚改稿改到两点，最后只删掉了一句话。
  • 我刷到一个同龄女生的视频，突然嫉妒到不想承认。
  1. 问题刺痛型
  • 你有没有发现，你越想把自己准备好，越开始不了？
  • 你有没有一种很奇怪的感觉：明明什么都没发生，但你就是很累？
  • 你是不是也会因为别人过得平静，而不是别人过得成功，感到嫉妒？

坏开头的特点：

  • 太大：关于人生，我想说几句。
  • 太空：每个人都要学会成长。
  • 太装：真正清醒的女人，从不……
  • 太像 AI：在当今快节奏社会中，很多女性面临……
  • 太像课堂：今天我们来聊一个话题。

第一句的标准：

观众刷到时，应该觉得“她怎么知道我这样”，而不是“她又要讲道理了”。

Chapter 8

正文怎么写：先让人相信，再给人解释

正文结构

一条口播正文最常见的失败，是太早讲道理。

正确顺序应该是：

具体场景 -> 情绪反应 -> 追问原因 -> 旧解释失效 -> 新解释出现 -> 小行动 / 许可

你可以使用三种基础结构。

8.1 60 秒结构：适合冷启动测试

第 1 句：隐藏痛点。

第 2-3 句：具体场景。

第 4-6 句：承认真实情绪。

第 7-9 句：重新解释。

最后 2 句：给观众许可或小动作。

适合主题：内耗、拖延、害怕开始、嫉妒、普通女孩做账号、低能量。

优点：容易拍、容易测试、成本低。

缺点：深度有限，不能承载复杂人生故事。

8.2 90 秒结构：最适合新手主力使用

开头 0-3 秒：说出一个隐蔽痛点。

3-25 秒：讲一个具体场景。

25-50 秒：说明你以前怎么误解它。

50-75 秒：给出新的理解。

75-90 秒：落到一个温和但明确的行动。

适合主题：女性成长、情绪自救、自我接纳、账号启动记录。

优点：既有细节，又能讲清一个转弯。

8.3 2-4 分钟结构：只给有强故事链的选题

不要因为对标账号能讲 3-6 分钟，你也立刻讲 3-6 分钟。长视频能成立，是因为它有足够多的决策点、反转点、细节证据和情绪升级。

长视频必须具备：

  • 一个真正复杂的问题。
  • 至少 3 个具体场景或人生节点。
  • 每个场景都推进理解，而不是重复表达。
  • 结尾有一个足够有力量的重新解释。

适合主题：重要人生选择、家庭影响、长期身体 / 精力限制、创作道路、关系模式。

新手建议：前 30 天主要做 45-90 秒，少量测试 2 分钟，不要一开始追求长篇大论。

Chapter 9

结尾怎么写：给许可，不要喊口号

结尾方式

这个账号的结尾，不应该像：

  • “所以女生一定要强大起来。”
  • “从今天开始逆袭。”
  • “关注我，带你走出内耗。”
  • “你一定会越来越好。”

更适合的是：

  • 也许今天不用马上变好，先别再继续怪自己。
  • 如果你也有这种感觉，可以先把这条留给自己。
  • 我们不用一下子跑起来，先把那个真正想要的东西承认下来。
  • 你已经撑到这里了，这本身不是一件很轻的事。
  • 明天不用证明你很厉害，先完成一个最小的动作。

结尾的作用不是“升华”，而是让观众从视频里安全落地。

好的结尾应该让人想：

  • 我被理解了。
  • 我可以保存这条。
  • 我可以转给一个人。
  • 我今天可以少攻击自己一点。
Chapter 10

拍摄标准：画面要真，不要假精致

拍摄标准

这类账号最重要的视觉感，不是贵、不是大片、不是过度美颜，而是：可信、亲近、稳定、能听清。

基础配置：

  • 手机竖屏 9:16。
  • 1080p 以上。
  • 固定机位，不要手持乱晃。
  • 尽量自然光，或者一盏柔和补光灯。
  • 背景干净但不要像样板间。
  • 收音清楚，有条件用领夹麦。
  • 每条至少拍 2-3 遍完整口播。

适合场景：

  • 卧室桌边。
  • 窗边。
  • 沙发 / 床边，但不要太随便。
  • 书桌前。
  • 安静咖啡馆角落。
  • 夜晚小灯下。
  • 简单户外走路，但收音必须过关。

不建议：

  • 过度磨皮美颜，失去真实表情。
  • 太像直播带货背景。
  • 背景过乱，让人分心。
  • 一上来就大量摆拍哭泣。
  • 随机风景 B-roll 堆砌。
  • 过度精致导致“她不像会说这种痛苦的人”。

镜头距离：

  • 情绪口播：脸部近景或半身。
  • 需要动作证明：半身 / 全身 / 桌面俯拍。
  • 讲账号过程：可以拍脚本、电脑、笔记、失败素材。

表演标准：

  • 不要像背课文。
  • 不要像演讲。
  • 不要每句话都很用力。
  • 允许停顿，但不要拖沓。
  • 允许眼神有一点不确定，但核心句要说清楚。
  • 像给一个信任的朋友讲，而不是向全网证明自己。

拍摄前检查：

  1. 这条稿子我真的能说吗？
  2. 有没有一句话让我觉得太假？删掉。
  3. 有没有一句话太像别人的表达？改掉。
  4. 第一遍先不拍，读出来听听。
  5. 正式拍 2-3 遍，不要每错一句就从头崩溃。
  6. 拍完立刻标记最好的一条。
Chapter 11

剪辑标准：不要把真诚剪没了

剪辑标准

这个账号的剪辑目标：清楚、稳定、让人愿意听下去。

不是：刺激、花哨、强节奏、模板感。

剪辑原则：

  1. 前 3 秒必须紧。

开头不要有咳嗽、调整姿势、眼神飘、找状态。

  1. 字幕必须清楚。

这类视频高度依赖语言，字幕是内容的一部分，不是装饰。

  1. 保留必要停顿。

情绪转弯处可以留一点呼吸，不要剪成机械连珠炮。

  1. B-roll 只在有意义时出现。

例如讲写稿，就出现手写稿；讲房间，就出现房间角落；讲拍摄失败，就出现失败片段。

不要随便放天空、花、街景、咖啡杯。

  1. 音乐可有可无。

如果用，音量要低，不要廉价煽情。

  1. 不要过度包装。

女性成长口播最怕变成“AI 情绪短片模板”。

字幕标准：

  • 一屏最多 1-2 行。
  • 字体大，手机上能看清。
  • 白色 / 暖白色 + 阴影或描边。
  • 关键词可以轻微强调，但不要满屏动画。
  • 不要为了押韵改内容。

剪辑交付给工程师 / AI 时，创作者需要提供：

  • 最好的一条主口播。
  • 可用 B-roll。
  • 哪一句是重点。
  • 哪些片段不能用。
  • 是否允许保留卡顿 / 笑场 / 犹豫。
Chapter 12

发布包装：标题、封面、文案、评论区

发布包装

发布包装不是骗点击，而是帮观众在 1 秒内知道“这条和我有关”。

12.1 封面文字

封面要短，最好 6-12 个字，像一个情绪标签或问题。

适合：

  • 你不是懒
  • 先别怪自己
  • 普通女孩先出现
  • 我也害怕被看见
  • 嫉妒不是坏
  • 别急着变好
  • 能撑住也算勇敢
  • 你不是太敏感

不适合：

  • 女性成长必看
  • 三分钟教你逆袭
  • 顶级清醒女生
  • 彻底摆脱内耗
  • 你必须明白的人生真相

12.2 标题

标题可以更完整一点，但仍然要像人话。

方向：

  • 有时候你不是不努力，你只是太怕被看见。
  • 普通女孩做账号，先别急着把自己变高级。
  • 我终于承认，我嫉妒的不是成功，而是平静。
  • 如果你也一直在准备，却迟迟不敢开始。
  • 低能量的人，也可以用很慢的方式把事做完。

12.3 正文 caption

正文不要重复整段口播。可以写一句补充：

  • 这条是写给还没开始、但已经责怪自己很久的人。
  • 我也在练习，不把每一次不完美都当成失败。
  • 先出现，再慢慢变好。
  • 如果你也有类似感觉，可以先把它留给自己。

12.4 评论区引导

不要硬求互动。

适合：

  • 你最近也有这种感觉吗？
  • 如果你愿意，可以留一句：你正在练习什么？
  • 这条先留给还在准备开始的人。
  • 你有没有一个一直想做但不敢开始的东西？

不适合：

  • 点赞关注不迷路。
  • 评论区告诉我你是哪一种女人。
  • 不点赞你一定会后悔。
Chapter 13

三方协同：创作者、AI、工程师分别做什么

三方职责

这个项目不能靠一个人蛮干。三方必须边界清楚。

13.1 创作者负责什么

创作者负责“真实”和“表达”。

必须做：

  1. 提供真实经历。
  2. 确认哪些话能说，哪些话不能说。
  3. 阅读并修改 AI 初稿，让它变成自己的话。
  4. 出镜拍摄。
  5. 对最终视频做发布前确认。
  6. 发布后看一部分评论，反馈真实感受。

不需要独自承担：

  • 全部对标研究。
  • 视频采集和转写。
  • 系统文件维护。
  • 数据表格整理。
  • 技术工具调试。
  • 所有剪辑自动化。

创作者每日最小任务：

  • 记录 3 条真实感受。
  • 读 1 条脚本并标出“不像我”的句子。
  • 如果有已批准脚本，拍 2-3 条可选素材。

创作者每周任务：

  • 提供 10-20 条经历碎片。
  • 拍摄 3-5 条视频素材。
  • 参加一次复盘，回答：哪条最像我？哪条我说起来最别扭？哪条评论让我有反应？

13.2 AI 负责什么

AI 负责“拆解、生成、检查、复盘”。

必须做：

  1. 读取项目定位、风格、禁区、内容标尺。
  2. 分析对标视频：钩子、结构、情绪、证据、结尾。
  3. 抽象选题，标记不能复制的部分。
  4. 根据创作者经历重写原创脚本。
  5. 给 30s / 60s / 90s 或主版本脚本。
  6. 做内容标尺检查。
  7. 生成拍摄计划、剪辑计划、发布包。
  8. 发布后整理数据和复盘建议。

AI 不允许做：

  • 编造创作者没经历过的故事。
  • 把对标账号原句改几个字当原创。
  • 为了“爆”而牺牲创作者边界。
  • 使用外部评分公式替代项目内容标尺。
  • 把心理 / 医疗 / 法律判断说得过度确定。

AI 的核心产物：

  • source.md
  • transcript.md
  • analysis.md
  • script.md
  • shooting-plan.md
  • editing-plan.md
  • publish-pack.md
  • review.md

13.3 工程师 / 主理人负责什么

工程师 / 主理人负责“系统让事情持续发生”。

必须做：

  1. 维护项目文件、脚本、仓库、目录结构。
  2. 采集 / 导入对标视频元数据。
  3. 管理转写、ASR、视频下载、原始素材路径。
  4. 保证敏感数据不进入 Git。
  5. 组织队列：哪些视频已处理，哪些待处理。
  6. 维护内容标尺、主题库、成功模式库。
  7. 监督 AI 不偏离账号定位和原创边界。
  8. 收集发布数据，建立复盘节奏。
  9. 把创作者从技术负担中解放出来。

工程师 / 主理人不是创作者的老板，也不是替创作者决定人生的人。

他的角色是系统协调者：让创作者专注表达，让 AI 高效产出，让流程能复用、能追踪、能迭代。

Chapter 14

系统协作引导：系统、工程师、创作者怎样配合

系统协作边界

这一章要直接告诉你：AI-IP-Studio 这个系统到底能帮你做什么，工程师 / 主理人能帮你做什么，哪些事情不能外包，哪些事情可以由你和系统一起做，再由你决定分工。

你不用把这个系统理解成一个“帮你写稿的软件”。

更准确地说，它是一个个人 IP 内容制片系统：它把对标研究、选题拆解、脚本生产、拍摄规划、剪辑规划、发布包装、数据复盘这些事情，变成一个可追踪、可复用、可迭代的生产流程。

它的目标不是把你替换掉，而是把你从混乱、重复、技术琐事和盲目试错中解放出来，让你把力气集中在最重要的事情上：真实经验、个人判断、镜头表达、长期选择。

14.1 这个系统能为你做什么

系统能做的，是把“做账号”这件很大的事拆小、拆清楚、拆成每天可以执行的任务。

它可以帮你：

  1. 找到值得学习的对标账号和视频。
  2. 把对标视频下载、转写、整理成可分析材料。
  3. 分析一条视频为什么可能有效：开头、结构、情绪、证据、转折、结尾。
  4. 从对标视频里提炼“可学习结构”，并标记“不能复制的部分”。
  5. 建立选题库，不让你每天凭空想选题。
  6. 根据你的真实经历，生成适合你说的脚本初稿。
  7. 给脚本做内容标尺检查：值不值得拍、像不像你、风险在哪里。
  8. 生成拍摄计划：在哪拍、怎么拍、拍几遍、需要哪些 B-roll。
  9. 生成剪辑计划：节奏、字幕、重点句、封面、标题、caption。
  10. 发布后整理数据，判断下一条应该继续、修改、暂停还是放弃。
  11. 把每次产出的材料保存成项目文件，避免经验散落在聊天记录里。
  12. 帮你发现长期模式：你适合讲什么、观众回应什么、你在哪些话题上最自然。

系统最擅长的是：整理复杂信息、重复执行流程、分析结构、生成备选方案、提醒风险、形成复盘闭环。

系统不擅长，也不应该替你做的是：代替你成为一个真实的人。

14.2 工程师 / 主理人能为你做什么

这里的工程师 / 主理人，就是帮你把系统跑起来的人。

他能做的不是“命令你怎么成为博主”，而是让整个内容生产过程更稳定、更省力、更不靠临时状态。

工程师 / 主理人可以做：

  1. 搭建和维护项目系统。

包括文件夹、模板、脚本、数据表、工作流、自动化工具。

  1. 做对标资料采集。

包括账号视频列表、指标、视频文件、音频、转写、评论数据等。

  1. 管理素材和文件。

让每条视频从来源到脚本到拍摄到复盘都有路径，不丢、不乱、不靠微信聊天记录找。

  1. 调试工具。

比如视频下载、ASR 转写、剪辑草稿、字幕、数据导出、网页工作台。

  1. 维护内容生产队列。

哪些视频已分析，哪些选题待写，哪些脚本待拍，哪些成片待复盘。

  1. 帮你把 AI 用起来。

他可以把你的经历、对标材料、项目规则喂给 AI，让 AI 产出更符合你的脚本和计划。

  1. 建立复盘机制。

记录每条视频的数据、评论、判断误差、下一步建议，而不是发完就忘。

  1. 保护边界和安全。

不把隐私、cookie、原始敏感素材、没确认的私人故事随便放进公开文件或提交记录。

  1. 做流程优化。

如果某一步太慢、太重复、太容易卡住，他负责想办法让它更省力。

  1. 帮你看长期趋势。

不用一条视频的好坏吓唬你，而是看 5 条、10 条、20 条之后到底什么在重复发生。

工程师 / 主理人的价值，是把账号从“今天有感觉就发一条”变成“稳定生产、稳定记录、稳定迭代”。

14.3 工程师 / 主理人不能为你做什么

有些事情，工程师再强也不能替你做。

他不能：

  1. 替你拥有真实经历。

他可以帮你整理经历，但不能编造你没经历过的人生。

  1. 替你决定你是谁。

账号定位可以一起讨论，但最终你愿意成为什么样的人、被别人怎样看见，必须由你确认。

  1. 替你判断哪些伤口能公开。

他可以提醒风险，但不能逼你讲任何你还没准备好承担后果的事情。

  1. 替你在镜头前成立。

脚本再好，最后仍然要通过你的声音、眼神、停顿和状态。

  1. 替你感受“不像我”。

AI 和工程师都能判断风格，但只有你最知道某句话说出口时是不是假。

  1. 替你承受发布后的反馈。

数据和评论可以一起看，但被看见的感受、被误解的难受、被喜欢的压力，最终发生在你身上。

  1. 替你选择长期方向。

他可以给建议、给证据、给趋势，但不能替你决定你未来半年要不要继续做这个表达方向。

  1. 替你把内容变成灵魂。

技术只能提高效率，不能制造真实。

如果工程师做得太多，账号会变成一个“系统产物”，而不是你的个人 IP。

如果创作者完全不参与思考，系统会越来越顺，但内容会越来越空。

14.4 AI 不能为你做什么

AI 很有用，但必须放在正确的位置。

AI 可以给你很多版本，但它不能替你判断哪一版最像你。

AI 可以把痛苦写得很漂亮，但它不知道你愿不愿意公开这个痛苦。

AI 可以从对标账号里提炼结构，但它不知道这个结构套在你身上会不会假。

AI 可以模拟口语，但它不能替你在镜头里说出口。

AI 可以帮你想标题，但它不知道你愿不愿意被这个标题定义。

AI 最容易犯的错误是：把一切都写得太完整、太顺、太像一个成熟博主。

而新账号最需要的，有时候恰恰不是成熟，而是可信。

所以你使用 AI 的正确方式不是：“帮我写一条爆款。”

而是：

  • 帮我把这个感受拆清楚。
  • 帮我找 3 个不同开头。
  • 帮我把这段话改得更像口播。
  • 帮我检查哪里像鸡汤。
  • 帮我检查哪里像对标账号。
  • 帮我给这条视频做拍摄计划。
  • 帮我根据数据判断下一条怎么改。

AI 是你的编导、助理、整理者和第二大脑，但不是你的替身。

14.5 最重要的事情，必须由创作者自己想

这个账号最核心的部分，必须由你自己参与思考。

尤其是下面这些问题，不能完全交给 AI，也不能完全交给工程师。

  1. 我到底想让别人怎样认识我？

不是“我要做女性成长博主”这么简单。

而是：

  • 我是温柔的，还是锋利的？
  • 我是陪伴型的，还是分析型的？
  • 我愿意暴露脆弱到什么程度？
  • 我想被当成朋友、姐姐、同龄人，还是一个正在路上的普通人？
  1. 哪些经历是我可以讲的？

你要自己决定：

  • 哪些可以真实讲。
  • 哪些只能改写后讲。
  • 哪些现在不能讲。
  • 哪些涉及别人隐私，不能讲。
  • 哪些讲出来会让我后悔。
  1. 我的真实感受是什么？

AI 可以帮你把情绪整理得好听，但你要先给它真实材料。

你需要诚实回答：

  • 我是真的这样想，还是觉得这样说比较容易火？
  • 我是真的被这件事触动，还是只是在模仿别人？
  • 我说这句话的时候，是舒服、害怕、羞耻，还是完全无感？
  1. 我愿意长期讲什么？

有些题一条会火，但你不想长期成为那种人。

比如：

  • 你愿不愿意长期讲关系创伤？
  • 你愿不愿意长期讲女性成长？
  • 你愿不愿意长期讲做账号的过程？
  • 你愿不愿意让观众看到你的不成熟？
  • 你愿不愿意被某个标签绑定？

这些都不是技术问题，是自我选择问题。

  1. 哪些内容会消耗我？

创作者必须学会识别：

  • 哪些话题拍完让我更轻松。
  • 哪些话题拍完让我崩溃。
  • 哪些评论我能承受。
  • 哪些误解会让我很久缓不过来。
  • 哪些选题虽然有流量，但不值得我反复消耗自己。

一个账号要长期做，不能只问“什么会火”，还要问“什么是我能持续承担的”。

14.6 哪些事系统能做，你也能做，由你决定怎么分工

有些工作不是非要谁做，可以按你的状态、兴趣和时间分配。

你可以自己做，也可以交给 AI / 工程师先做初稿，再由你决定。

  1. 选题发现

你可以做：记录生活里的痛点、刷到评论时截图、看到对标视频时说“这个我有感觉”。

系统可以做：批量分析对标视频、从数据里挑高潜力主题、整理 topic-library。

推荐分工：系统先给候选，你从里面挑“我真的有话说”的。

  1. 开头选择

你可以做：凭直觉选最像你、最想说的一句。

AI 可以做：给 5-10 个不同类型开头。

推荐分工：AI 多给，你少选；不要让 AI 直接决定最终开头。

  1. 脚本初稿

你可以做：自己口述一版粗糙想法，哪怕很乱。

AI 可以做：把你的口述整理成结构清楚的脚本。

推荐分工：你给真实原料，AI 做初稿，你再改回自己的话。

  1. 例子选择

你可以做：提供真实场景。

AI 可以做：提醒哪里缺例子、建议可以换成哪类场景。

推荐分工：例子必须来自你；AI 只能帮你想“需要哪种例子”。

  1. 标题和封面

你可以做：判断哪个标题不会让你觉得尴尬。

AI 可以做：生成多个标题、封面文字、caption。

推荐分工：AI 生成 10 个，你选 1-2 个，再由工程师记录发布效果。

  1. 拍摄方式

你可以做：决定自己舒服的场景、妆造、镜头距离。

AI 可以做：给镜头建议、B-roll 清单、提词器版本。

工程师可以做：安排素材路径、检查格式、协助后期。

推荐分工：你的舒适感优先，系统只做辅助。

  1. 评论复盘

你可以做：说哪些评论让你有感受、哪些让你不舒服。

AI 可以做：整理高频评论、分类正负反馈、提炼下一步选题。

工程师可以做：记录数据、保存评论样本、更新复盘文件。

推荐分工：系统看结构，你看情绪；两者都重要。

  1. 账号方向调整

你可以做：说自己想更靠近哪种表达，想远离哪种表达。

AI / 工程师可以做：用数据和内容样本说明哪条路线更稳。

推荐分工：系统提供证据，你做最终选择。

14.7 系统最需要你提供什么数据

系统不是越多数据越好，而是越真实、越具体、越可判断越好。

最需要你提供 8 类数据。

  1. 真实经历碎片

格式：发生了什么、我当时怎么想、我为什么在意、是否能公开。

  1. 情绪词

不要只说“还行”“有点难受”。

尽量具体：嫉妒、羞耻、委屈、害怕、烦躁、空、麻木、松了一口气、不服、想逃、想证明、想被看见。

  1. 不能说清单

包括：不能公开的人、不能讲的关系、不能提的地点、不能出现的画面、不能被平台误解的表达。

  1. 可说程度

每个经历最好标：

  • 可以真实讲。
  • 可以改写讲。
  • 只能抽象讲。
  • 现在不能讲。
  1. 口语偏好

你平时会说什么、不说什么。

比如你会不会说“内耗”“清醒”“搞事业”“自救”“普通女孩”“不配感”。

AI 写得再好，如果不是你的词，也要改掉。

  1. 拍摄条件

包括：能在哪拍、什么时候能拍、有没有麦克风、是否能拍全身、是否能拍生活 B-roll、是否能接受素颜 / 淡妆 / 近景。

  1. 发布后的感受

包括：哪条你最喜欢、哪条你后悔、哪条评论伤到你、哪条数据让你意外、哪条你愿意继续做。

  1. 现实时间和精力

系统需要知道你每周到底能投入多少。

比如：一周能拍几次、每次能拍多久、剪辑是否交给别人、是否能每天记录素材。

没有这些数据，AI 会写出看起来很好但你根本执行不了的计划。

14.8 工程师最需要你交付什么内容

如果你想让工程师高效帮你，你给他的材料要尽量清楚。

最有用的交付不是一句“你帮我看看”，而是以下这些。

  1. 经历素材

格式：

  • 事件：
  • 当时感受：
  • 我真正想说的是：
  • 能否公开：
  • 涉及隐私：
  1. 对标视频反馈

不要只说“我喜欢这条”。

要尽量说：

  • 我喜欢它的开头。
  • 我喜欢它把一个小事讲成大问题。
  • 我喜欢它的语气。
  • 我不喜欢它太长。
  • 这个话题我也有类似经历。
  • 这个结构可以学，但故事不能学。
  1. 脚本反馈

不要只说“怪怪的”。

要标：

  • 第几句不像我。
  • 哪个词我不会说。
  • 哪个经历不是我的。
  • 哪里太像鸡汤。
  • 哪里太像别人。
  • 哪里我愿意换成自己的例子。
  1. 拍摄素材

最好按项目约定保存：

  • 主口播第 1 遍 / 第 2 遍 / 第 3 遍。
  • 可用 B-roll。
  • 不可用片段说明。
  • 最满意的一遍。
  • 重点句时间点，如果你记得。
  1. 发布数据

截图可以，但最好包含：

  • 发布时间。
  • 播放。
  • 点赞。
  • 评论。
  • 转发。
  • 收藏。
  • 涨粉。
  • 完播率 / 平均观看时长，如果有。
  • 高赞评论截图。
  1. 主观复盘

工程师能看数据，但你要补充感受：

  • 这条我拍得舒服 / 不舒服。
  • 我觉得这条最像我 / 不像我。
  • 我愿意继续做这个方向 / 不想继续。
  • 评论让我觉得被理解 / 被误解。

工程师越清楚这些，他越能帮系统做出准确判断。

14.9 三方最高效的协作方式

最高效的协作，不是所有人都参与所有事，而是每个阶段只让最该负责的人做决定。

推荐节奏：

  1. 工程师 / AI 先给候选，不让创作者从空白开始。
  2. 创作者只做关键判断：像不像我、能不能说、愿不愿意拍。
  3. AI 根据判断改稿，不让创作者陷入文字细节里出不来。
  4. 工程师把文件、素材、版本、数据管理好，不让流程靠记忆。
  5. 发布后系统先整理数据，创作者再补充主观感受。
  6. 每周只做一次方向复盘，不要每天因为数据波动改定位。

三方协作的关键原则：

  • 系统负责“给选择”，创作者负责“做选择”。
  • 工程师负责“让流程稳定”，创作者负责“让内容真实”。
  • AI 负责“把材料变成方案”，创作者负责“判断方案是不是自己”。
  • 数据负责“提醒我们哪里错了”，但不能替代创作者的长期意愿。

14.10 如果不知道该谁做，就按这个规则分

凡是涉及真实人生、公开边界、身份选择、长期方向的，由创作者最终决定。

凡是涉及采集、转写、整理、命名、归档、工具、版本、数据的，由工程师 / 系统负责。

凡是涉及分析、生成、改写、检查、总结、备选方案的，由 AI 先做。

凡是涉及“像不像我”“我愿不愿意说”“我说出口舒不舒服”的，由创作者判断。

凡是涉及“这个流程能不能长期跑”“素材会不会丢”“数据能不能复盘”的，由工程师判断。

凡是涉及“这条内容能不能被观众理解”“结构有没有问题”“风险在哪里”的，由 AI 和工程师先判断，创作者最终确认。

简单说：

  • 你的生活，你决定。
  • 系统的流程，工程师负责。
  • 内容的初稿，AI 负责。
  • 最终能不能说，你决定。
  • 最终发不发布，三方看完后由你和主理人确认。

14.11 创作者不要怎样使用这个系统

不要把系统当成许愿池：

  • 帮我做一个能火的账号。
  • 帮我写一个爆款。
  • 你们决定我说什么。
  • 我什么都不提供，你们帮我想。

这样系统会产出很多看起来完整但没有灵魂的东西。

也不要把系统当成考试老师：

  • 我是不是不适合？
  • 我这条是不是失败？
  • 数据不好是不是我不行？

系统不是来审判你的。系统是来帮助你把每一次尝试变成下一次的经验。

更好的使用方式是：

  • 我这里有一个真实经历，帮我判断能不能变成选题。
  • 这条脚本有几句不像我，帮我改口语一点。
  • 我想拍这条，但担心太暴露，帮我降风险。
  • 这条发布后收藏高但评论少，帮我判断下一条怎么延展。
  • 我这一周状态不好，帮我把任务拆到最小。

14.12 一次高质量协作应该长什么样

低质量协作：

创作者：我想做女性成长，你们帮我写几条。

AI：给 10 条泛泛脚本。

创作者：感觉不太像我。

工程师：不知道哪里不像，流程卡住。

高质量协作：

创作者：我昨天因为拍视频没拍好，突然觉得自己不配做账号。这个情绪可以讲，但不要显得太惨。我想表达的是：有时候不是不想开始，是太害怕被看见。可以公开，但不要提具体人。

AI：根据这个经历给 3 个选题角度、5 个开头、1 条 90 秒脚本，并标出风险。

工程师 / 主理人：把这条建立成 output 包，记录来源、脚本版本、拍摄计划和发布后复盘位置。

创作者：选第 2 个开头，删掉 3 句不像自己的话，换成自己的真实场景，拍 3 遍。

AI / 工程师：生成剪辑计划、标题、封面、caption，发布后记录数据。

复盘：这条是 repeat、revise、park 还是 stop。

这就是系统最希望形成的工作方式。

14.13 最后提醒：系统越强，你越要保留自己的判断

当系统越来越完整，AI 越来越会写，工程师越来越能自动化，你反而更要保留自己的判断。

因为个人 IP 最怕的不是效率低，而是效率很高地生产出“不像你”的内容。

你永远要问：

  • 这件事我真的在意吗？
  • 这句话我真的会说吗？
  • 这个经历真的属于我吗？
  • 这个方向我愿意被长期看见吗？
  • 这条视频即使不火，我会不会觉得它仍然值得？

如果答案都是否定的，那系统再顺，也应该停下来重想。

系统是为了让你更稳定地成为你自己，而不是更快地变成另一个人。

Chapter 15

标准工作流程：从一个灵感到一条发布视频

完整生产流程

以下是本项目推荐的完整流程。

15.1 阶段 A：对标输入

负责人：工程师 + AI

创作者参与：少量判断

步骤：

  1. 工程师收集目标账号 / 视频。
  2. 保存元数据：链接、平台、作者、标题、指标、发布时间。
  3. 下载或导入视频 / 音频。

  • repeat：值得继续做同类。

  • 做内容禁区清单。

  • 写 20 条真实经历碎片。
  • 录 1 条不发布的自我介绍口播。
  • 确认哪些经历可以讲，哪些不能讲。
  • 确认镜头舒适边界。

第 1 周：低成本试拍

目标：拍出来，不追求完美。

产出：

  • 5 个选题。
  • 5 条脚本。
  • 3 条正式拍摄。
  • 1-3 条发布。

推荐主题：

  • 害怕开始。
  • 普通女孩先出现。
  • 不是懒，是太怕失败。
  • 嫉妒别人平静。
  • 低能量也可以完成一点。

重点：

  • 测试创作者在镜头前的自然表达。
  • 测试哪种句子她说起来最顺。
  • 不要一开始追求复杂剪辑。

第 2 周：建立可重复结构

目标：找到 2-3 个可持续栏目。

产出：

  • 每周 3-5 条发布。
  • 每条都做简短复盘。
  • 建立“开头库”。
  • 建立“真实经历库”。

测试栏目：

  1. 我也是后来才明白。
  2. 普通女孩做账号记录。
  3. 先别急着怪自己。
  4. 一件小事让我意识到。
  5. 读到 / 听到一句话以后。

重点：

  • 看收藏高的是哪类。
  • 看评论多的是哪类。
  • 看创作者最能持续说的是哪类。

第 3 周：加强差异化

目标：不要变成普通情绪号。

加入：

  • 更具体的个人经历。
  • 更清楚的创作者身份：正在做账号的人、正在练习表达的人、正在变稳定的人。
  • 更强的过程透明：写稿、拍摄、失败、修改、复盘。

可以尝试：

  • 回应一个真实评论。
  • 展示一次改稿过程。
  • 拍一条“我为什么不想装高级”。
  • 拍一条“我今天只完成了最小动作”。

第 4 周：复盘和收窄

目标：决定下一阶段主线。

工程师 / AI 输出月度报告：

  • 哪 3 条数据最好。
  • 哪 3 条最像创作者。
  • 哪类选题收藏高。
  • 哪类选题评论多。
  • 哪类选题创作者说起来最自然。
  • 哪些表达风险高。
  • 下个月保留 / 放弃 / 加强什么。

创作者回答：

  • 哪条我最愿意继续做？
  • 哪条我最不想再拍？
  • 哪些评论让我觉得被理解？
  • 哪些话我说出来不舒服？
  • 我下个月想把自己往哪个方向推一点？

新手最容易犯的 20 个错误

新手错误

  1. 一上来就想做成大博主质感。
  2. 每条都想讲人生真理。
  3. 没有具体场景，全是抽象感悟。
  4. 直接复制对标账号的故事和比喻。
  5. 把 AI 稿子不改就读。
  6. 开头太像标题党。
  7. 结尾太像鸡汤。
  8. 用“女人就应该”“你必须”这种命令语气。
  9. 没有自己的经历，却硬讲深刻话题。
  10. 过度解释，视频像作文。
  11. 拍摄太追求完美，导致迟迟不发布。
  12. 剪辑太花，把真诚剪没了。
  13. 每天换方向，账号没有稳定识别。
  14. 只看点赞，不看收藏、转发、评论结构。
  15. 被一条差数据打崩。
  16. 过早商业化，伤害信任。
  17. 把个人伤口讲得太多，自己承受不了。
  18. 把观众当学生，而不是当朋友。
  19. 只追热点，不建立自己的问题库。
  20. 没有复盘，发完就忘。
Chapter 18

内容安全和原创边界

安全与原创

必须遵守：

  1. 不说不是自己的故事。
  2. 不把对标账号的独特表达换几个字拿来用。
  3. 不复制对标视频的例子顺序。
  4. 不冒充专业心理、医学、法律建议。
  5. 不承诺观众一定会变好、变强、涨粉、赚钱、被爱。
  6. 不为了流量制造极端性别对立。
  7. 不公开第三方隐私。
  8. 不讲创作者本人尚未准备好承担后果的私人细节。

原创转换至少改变 3 层：

  • 观点。
  • 例子。
  • 观众场景。
  • 结论。
  • 情绪立场。
  • 结构顺序。
  • 行动建议。

每条从对标视频来的内容，都要问：

  • 我学的是结构，还是在偷她的人生？
  • 这个例子是不是我的？
  • 这句话如果原作者刷到，会不会觉得太像？
  • 创作者说出来是否真实？

Chapter 21

最终原则

长期原则

这个账号要长期做下去，最重要的不是“每条都完美”，而是守住以下原则：

  1. 真实比高级重要。
  2. 具体比深刻重要。
  3. 可持续比爆一次重要。
  4. 个人经验比空泛观点重要。
  5. 结构学习可以，人生不能复制。
  6. AI 是制片工具，不是灵魂替身。
  7. 工程师是流程保护者，不是内容独裁者。
  8. 创作者是账号的中心，她的边界必须被尊重。
  9. 冷启动不要假装自己已经懂流量。
  10. 每一条视频都要让观众更清楚一点，而不是更焦虑一点。

你不需要一开始就是一个成熟博主。

你只需要先成为一个能够稳定出现、稳定表达、稳定复盘的人。

账号不是一天长出来的。账号是一条一条视频，把你从“害怕被看见的人”，训练成“可以把自己说清楚的人”。

这才是这个项目真正要帮你完成的事。
